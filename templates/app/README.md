## Auth Setup

This app requires Clerk authentication. Before running:

1. Copy `.env.example` to `.env.local` in the monorepo root (or this directory)
2. Fill in your Clerk keys
3. Configure Clerk Dashboard — see `docs/AUTH-SETUP.md` for the full setup guide

### Adding this module's permission in Clerk

After running the scaffold script, add the permission `org:__APP_SLUG__:access` in the Clerk Dashboard:

- **Clerk Dashboard → Configure → Roles & Permissions**
- Add permission: `org:__APP_SLUG__:access`
- Assign to roles: `org:admin` (always), and any other roles that should have access by default

## Extending the App

### Add a new route

1. Create a new directory under `src/app/(app)/`
2. Add a `page.tsx` (server component by default)
3. Register the route in the nav config (see Shell registry)

### Add a new component

1. Check `@repo/ui` first — it has 55+ components and 28+ patterns
2. If nothing fits, create it in `src/components/`

### Connect to real data

Replace the seed data in `src/lib/data.ts` with real queries. Server components can fetch directly; client components should use tRPC or SWR.
